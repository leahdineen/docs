## Overview

There are two ways to create a judge:

1. **With Detectors (Recommended)** - Specify detectors and their configurations directly
2. **With Criteria (Legacy)** - Specify evaluation criteria that auto-generate detector configs

## Authentication

This endpoint supports dual authentication:
- **Bearer Token**: Auth0 JWT token in the `Authorization` header
- **API Key**: Organization API key in the `X-API-Key` header

## Create Judge with Detectors (Recommended)

### Request Body

<ParamField body="name" type="string" required>
  The name of the judge. Must be between 1-200 characters.
</ParamField>

<ParamField body="description" type="string" required>
  A description of what this judge evaluates. Must be between 1-2000 characters.
</ParamField>

<ParamField body="content_type" type="string" required>
  The type of content this judge will evaluate.
  
  **Allowed values:** `video`, `image`, `audio`, `text`
</ParamField>

<ParamField body="detectors" type="array" required>
  An array of detector configurations. Must contain 1-50 detectors.
  
  <Expandable title="Detector Configuration">
    <ParamField body="detector_name" type="string" required>
      The name of the detector. Must be a valid detector name.
      
      See [Detector Configuration Reference](/api-reference/detectors/configuration-reference) for available detectors.
    </ParamField>
    
    <ParamField body="config" type="object" required>
      Configuration object for the detector. Required keys depend on the detector type.
      
      See [Detector Configuration Reference](/api-reference/detectors/configuration-reference) for detector-specific configurations.
    </ParamField>
  </Expandable>
</ParamField>

### Request Example with Detectors

```json
{
  "name": "Video Quality Judge",
  "description": "Evaluates video technical quality and content",
  "content_type": "video",
  "detectors": [
    {
      "detector_name": "video_specs",
      "config": {
        "min_width": 1920,
        "min_height": 1080,
        "min_duration_sec": 15,
        "max_duration_sec": 60,
        "min_fps": 24
      }
    },
    {
      "detector_name": "audio_noise",
      "config": {}
    },
    {
      "detector_name": "transcript_product",
      "config": {
        "brand_name": "Acme Corp",
        "product_name": "Super Widget"
      }
    }
  ]
}
```

### Example with Reference Content

For detectors that require reference content (like `video_matching`), first upload the reference content:

1. Upload reference content using `/v1/judges/reference_content`
2. Use the returned `gcs_uri` in the detector config:

```json
{
  "name": "Content Matching Judge",
  "description": "Matches videos against reference content",
  "content_type": "video",
  "detectors": [
    {
      "detector_name": "video_matching",
      "config": {
        "reference_video": "gs://judge-artifacts/org_123/reference_uploads/abc/reference.mp4"
      }
    },
    {
      "detector_name": "audio_matching",
      "config": {
        "reference_audio": "gs://judge-artifacts/org_123/reference_uploads/def/audio.mp3"
      }
    }
  ]
}
```

## Create Judge with Criteria (Legacy)

### Request Body

<ParamField body="name" type="string" required>
  The name of the judge. Must be between 1-200 characters.
</ParamField>

<ParamField body="description" type="string" required>
  A description of what this judge evaluates. Must be between 1-2000 characters.
</ParamField>

<ParamField body="content_type" type="string" required>
  The type of content this judge will evaluate.
  
  **Allowed values:** `video`, `image`, `audio`, `text`
</ParamField>

<ParamField body="criteria" type="array" required>
  An array of evaluation criteria for this judge. Must contain 1-50 criteria.
  
  <Expandable title="Criterion Object">
    <ParamField body="name" type="string" required>
      The name of the criterion. Must be between 1-200 characters.
    </ParamField>
    
    <ParamField body="description" type="string" required>
      A description of what this criterion evaluates. Must be between 1-1000 characters.
    </ParamField>
    
    <ParamField body="weight" type="integer" default="5">
      The weight of this criterion in the overall evaluation. Must be between 1-10.
    </ParamField>
  </Expandable>
</ParamField>

### Request Example with Criteria

```json
{
  "name": "Content Safety Judge",
  "description": "Evaluates video content for safety and compliance",
  "content_type": "video",
  "criteria": [
    {
      "name": "Violence Detection",
      "description": "Detects violent content in videos",
      "weight": 8
    },
    {
      "name": "Language Appropriateness",
      "description": "Checks for inappropriate language",
      "weight": 6
    }
  ]
}
```

## Response

### Response for Detector-based Creation

<ResponseField name="id" type="string">
  The unique identifier of the created judge
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the judge
  
  **Possible values:**
  - `pending` - Judge is being prepared (reference content preprocessing)
  - `ready` - Judge is ready to use
  - `failed` - Judge creation failed
</ResponseField>

### Response Example (Detectors)

```json
{
  "id": "judge_abc123",
  "status": "ready"
}
```

Note: If the judge includes detectors that require preprocessing (like `video_matching` or `audio_matching`), the status will be `pending` until preprocessing completes.

### Response for Criteria-based Creation

<ResponseField name="id" type="string">
  The unique identifier of the created judge
</ResponseField>

<ResponseField name="name" type="string">
  The name of the judge
</ResponseField>

<ResponseField name="description" type="string">
  The description of the judge
</ResponseField>

<ResponseField name="content_type" type="string">
  The content type this judge evaluates
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the judge
  
  **Possible values:** `pending`, `training`, `ready`, `failed`
</ResponseField>

<ResponseField name="training_progress" type="integer">
  Training progress percentage (0-100)
</ResponseField>

<ResponseField name="criteria" type="array">
  Array of created criteria objects
  
  <Expandable title="Criterion Response">
    <ResponseField name="id" type="string">
      The unique identifier of the criterion
    </ResponseField>
    
    <ResponseField name="name" type="string">
      The name of the criterion
    </ResponseField>
    
    <ResponseField name="description" type="string">
      The description of the criterion
    </ResponseField>
    
    <ResponseField name="weight" type="integer">
      The weight of this criterion
    </ResponseField>
    
    <ResponseField name="detector_configs" type="array">
      Array of detector configurations (for video judges)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the judge was created
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of when the judge was last updated
</ResponseField>

<ResponseField name="version" type="integer">
  The version number of this judge
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether this judge is currently active
</ResponseField>

## Error Responses

### 400 Bad Request
Invalid request body or validation errors

### 401 Unauthorized
Missing or invalid authentication

### 403 Forbidden
Insufficient permissions

### 500 Internal Server Error
Failed to create judge due to server error
