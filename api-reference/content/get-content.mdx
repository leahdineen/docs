---
title: 'Get Content'
api: 'GET /content/{content_id}'
description: 'Retrieve content details'
---

## Overview

Retrieves details of a specific content item including metadata and processing status.

## Path Parameters

<ParamField path="content_id" type="string" required>
  The unique identifier of the content
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Content identifier
</ResponseField>

<ResponseField name="name" type="string" required>
  Content name
</ResponseField>

<ResponseField name="description" type="string">
  Content description
</ResponseField>

<ResponseField name="file_type" type="string" required>
  Content type (text, video, audio, image)
</ResponseField>

<ResponseField name="size" type="integer" required>
  File size in bytes
</ResponseField>

<ResponseField name="status" type="string" required>
  Processing status
  - `uploaded` - Successfully uploaded
  - `processing` - Being processed
  - `ready` - Ready for evaluation
  - `error` - Processing failed
</ResponseField>

<ResponseField name="text_content" type="string">
  Text content (for text type)
</ResponseField>

<ResponseField name="file_path" type="string">
  Internal file path
</ResponseField>

<ResponseField name="file_url" type="string">
  External file URL
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Upload timestamp
</ResponseField>

## Examples

<CodeGroup>

```bash Request
curl -X GET https://api.judge.example.com/content/cnt_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>
