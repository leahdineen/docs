## Overview

The `language` detector analyzes spoken content to identify the language being used and validates it against a list of accepted languages. This ensures content meets language requirements for specific regions or audiences.

## Configuration

<ParamField body="config.accepted_languages" type="array" required>
  List of accepted language codes (ISO 639-1 format, e.g., ["en", "es", "fr"])
</ParamField>

## Example Configuration

```json
{
  "detector_name": "language",
  "config": {
    "accepted_languages": ["en", "es"]
  }
}
```

## Interpreting Results

- **detected_language**: ISO 639-1 code for the primary detected language
- **confidence**: Detection confidence score (0.0-1.0)
- **duration_percentage**: Percentage of content in each detected language
- **language_segments**: Time-stamped segments showing language changes
- **accepted_languages**: Use ISO 639-1 codes (en, es, fr, de, etc.)
