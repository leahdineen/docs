## Overview

Assesses whether the content meets general quality and value expectations.

## Configuration

This detector does not accept configuration keys.

## Example Configuration

```json
{
  "detector_name": "content_quality",
  "config": {}
}
```

## Result Schema

```json
{
  "detector_name": "content_quality",
  "pass_check": true,
  "score": 0.84,
  "rationale": "Content quality is acceptable. Quality score: 0.90/1.00. Provides clear, actionable guidance.",
  "metrics": {
    "engagement_farming_count": 0,
    "brain_rot_count": 0,
    "engagement_farming_instances": [],
    "brain_rot_instances": [],
    "overall_quality_score": 0.9,
    "has_genuine_value": true,
    "value_assessment": "Provides clear, actionable guidance.",
    "reasoning": "",
    "model": "gpt-4o-mini",
    "requirements": {
      "max_engagement_farming": 0,
      "max_brain_rot": 0,
      "confidence_threshold": 0.6
    }
  },
  "timestamps": [],
  "flagged_frames": []
}
```

## Interpreting Results

- **overall_quality_score**: Core quality signal used in the final score.
- **engagement_farming_count/brain_rot_count**: Issue counts against the configured limits.
- **has_genuine_value/value_assessment**: Summary judgment and reasoning.
