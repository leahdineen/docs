## Overview

The `language` detector analyzes spoken content to identify the language being used and validates it against a list of accepted languages. This ensures content meets language requirements for specific regions or audiences.

## Configuration

<ParamField body="config.accepted_languages" type="array" required>
  List of accepted language codes (ISO 639-1 format, e.g., ["en", "es", "fr"])
</ParamField>

## Example Configuration

```json
{
  "detector_name": "language",
  "config": {
    "accepted_languages": ["en", "es"]
  }
}
```

## Supported Language Codes

Common ISO 639-1 language codes:

- **en** - English
- **es** - Spanish  
- **fr** - French
- **de** - German
- **it** - Italian
- **pt** - Portuguese
- **ru** - Russian
- **ja** - Japanese
- **ko** - Korean
- **zh** - Chinese
- **ar** - Arabic
- **hi** - Hindi

## Language Detection Features

- **Primary Language** - The dominant language in the content
- **Secondary Languages** - Other languages detected with lower confidence
- **Confidence Scores** - Reliability of language detection (0.0-1.0)
- **Duration Analysis** - Percentage of content in each language
- **Segment-Level Detection** - Time-stamped language changes

## Use Cases

- **Regional Compliance** - Ensuring content meets local language requirements
- **Content Moderation** - Verifying content is in expected languages
- **Audience Targeting** - Confirming content matches target demographics
- **Platform Requirements** - Meeting language guidelines for social platforms

## Detector Type

**AI_TEXT** - Uses AI models for language detection and analysis
