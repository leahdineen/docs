---
title: 'Rollback Judge'
api: 'POST /judges/{judge_id}/rollback'
description: 'Rollback judge to a previous version'
---

## Overview

Rolls back a judge to a previous version. This is useful when a new version performs worse than expected or introduces issues.

## Path Parameters

<ParamField path="judge_id" type="string" required>
  Unique identifier of the judge
</ParamField>

## Request Body

<ParamField body="target_version" type="integer" required>
  Version number to rollback to
</ParamField>

<ParamField body="reason" type="string" required maxLength="500">
  Reason for the rollback
</ParamField>

## Response

<ResponseField name="judge_id" type="string" required>
  ID of the judge that was rolled back
</ResponseField>

<ResponseField name="previous_version" type="integer" required>
  Version number before rollback
</ResponseField>

<ResponseField name="current_version" type="integer" required>
  Version number after rollback
</ResponseField>

<ResponseField name="rollback_successful" type="boolean" required>
  Whether rollback completed successfully
</ResponseField>

<ResponseField name="rolled_back_at" type="datetime" required>
  Rollback timestamp
</ResponseField>

## Examples

<CodeGroup>

```bash Request
curl -X POST https://api.judge.example.com/judges/jdg_xyz789/rollback \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "target_version": 2,
    "reason": "Version 3 showing decreased accuracy in production"
  }'
```

```json Success Response (200)
{
  "judge_id": "jdg_xyz789",
  "previous_version": 3,
  "current_version": 2,
  "rollback_successful": true,
  "rolled_back_at": "2024-01-15T17:00:00Z"
}
```

</CodeGroup>
