## Overview

The `video_specs` detector validates video technical specifications against configurable thresholds. This heuristic detector checks dimensions, duration, frame rate, and aspect ratio to ensure videos meet quality requirements.

## Configuration

<ParamField body="config.min_width" type="integer">
  Minimum video width in pixels
</ParamField>

<ParamField body="config.max_width" type="integer">
  Maximum video width in pixels
</ParamField>

<ParamField body="config.min_height" type="integer">
  Minimum video height in pixels
</ParamField>

<ParamField body="config.max_height" type="integer">
  Maximum video height in pixels
</ParamField>

<ParamField body="config.allowed_aspect_ratios" type="array">
  List of allowed aspect ratios (e.g., ["16:9", "9:16", "1:1"])
</ParamField>

<ParamField body="config.min_duration_sec" type="number">
  Minimum video duration in seconds
</ParamField>

<ParamField body="config.max_duration_sec" type="number">
  Maximum video duration in seconds
</ParamField>

<ParamField body="config.min_fps" type="number">
  Minimum frames per second
</ParamField>

<ParamField body="config.max_fps" type="number">
  Maximum frames per second
</ParamField>

## Example Configuration

```json
{
  "detector_name": "video_specs",
  "config": {
    "min_width": 1920,
    "min_height": 1080,
    "min_duration_sec": 15,
    "max_duration_sec": 60,
    "min_fps": 24,
    "allowed_aspect_ratios": ["16:9", "9:16"]
  }
}
```

## Interpreting Results

The video_specs detector validates video technical specifications and returns:

- **Video Specs** - Actual video dimensions, aspect ratio, frame rate, and duration
- **Compliance Check** - Boolean validation against configured requirements (min/max width, height, etc.)
- **Aspect Ratio** - Calculated ratio and validation against allowed ratios
- **Duration** - Video length in seconds with validation against min/max duration requirements
- **Audio Sample Rate** - Audio technical specifications if present in the video
