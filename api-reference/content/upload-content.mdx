## Overview

Uploads content to the system for evaluation. Supports text, video, audio, and image files. Content can be provided as direct text, file upload, or URL.

## Request (Multipart Form)

<ParamField body="file" type="file">
  File to upload (for video, audio, image content)
</ParamField>

<ParamField body="text_content" type="string">
  Direct text content (for text type only)
</ParamField>

<ParamField body="file_url" type="string">
  URL to external content file
</ParamField>

<ParamField body="name" type="string" required>
  Name/title of the content
</ParamField>

<ParamField body="description" type="string">
  Description of the content
</ParamField>

<ParamField body="file_type" type="string" required>
  Content type
  - `text` - Text content
  - `video` - Video files (mp4, avi, mov)
  - `audio` - Audio files (mp3, wav, m4a)
  - `image` - Image files (jpg, png, gif)
</ParamField>

<ParamField body="metadata" type="json">
  Additional metadata as JSON string
</ParamField>

## Response

<ResponseField name="content_id" type="string" required>
  Unique identifier for uploaded content
</ResponseField>

<ResponseField name="name" type="string" required>
  Content name
</ResponseField>

<ResponseField name="file_type" type="string" required>
  Content type
</ResponseField>

<ResponseField name="size" type="integer" required>
  File size in bytes
</ResponseField>

<ResponseField name="status" type="string" required>
  Processing status
  - `uploaded` - Successfully uploaded
  - `processing` - Being processed
  - `ready` - Ready for evaluation
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Upload timestamp
</ResponseField>

<ResponseField name="file_path" type="string">
  Internal file storage path
</ResponseField>

<ResponseField name="file_url" type="string">
  External file URL if provided
</ResponseField>

## Examples

<CodeGroup>

```bash Text Upload
curl -X POST https://api.judge.example.com/content/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "text_content=This is sample text content to evaluate" \
  -F "name=Sample Article" \
  -F "description=Article about API documentation" \
  -F "file_type=text"
```

```json Success Response
{
  "content_id": "cnt_abc123",
  "name": "Sample Article",
  "file_type": "text",
  "size": 42,
  "status": "ready",
  "created_at": "2024-01-15T10:00:00Z",
  "file_path": null,
  "file_url": null
}
```

</CodeGroup>

<CodeGroup>

```bash Video Upload
curl -X POST https://api.judge.example.com/content/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@video.mp4" \
  -F "name=Product Demo Video" \
  -F "description=Product demonstration" \
  -F "file_type=video"
```

```json Success Response
{
  "content_id": "cnt_vid456",
  "name": "Product Demo Video",
  "file_type": "video",
  "size": 15728640,
  "status": "processing",
  "created_at": "2024-01-15T10:00:00Z",
  "file_path": "/storage/videos/cnt_vid456.mp4",
  "file_url": null
}
```

</CodeGroup>

<CodeGroup>

```bash Error: Invalid File Type
curl -X POST https://api.judge.example.com/content/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@document.exe" \
  -F "name=Invalid File" \
  -F "file_type=text"
```

```json Error Response (400)
{
  "error": {
    "code": "INVALID_FILE_TYPE",
    "message": "File type '.exe' is not supported"
  }
}
```

</CodeGroup>
