title: 'Database Health'
api: 'GET /health/database'
description: 'Detailed database health check'
---

## Overview

Returns detailed information about database connectivity and configuration. Useful for debugging connection issues in different environments.

## Response

<ResponseField name="status" type="string" required>
  Database health status
  - `healthy` - Database connection successful
  - `unhealthy` - Database connection failed
</ResponseField>

<ResponseField name="environment" type="string" required>
  Current environment (development, staging, production)
</ResponseField>

<ResponseField name="database_url" type="string" required>
  Sanitized database connection string (credentials hidden)
</ResponseField>

<ResponseField name="error" type="string">
  Error message if connection failed
</ResponseField>

## Examples

<CodeGroup>

```bash Success Response
curl -X GET https://api.getpique.ai/health/database
```

```json Response
{
  "status": "healthy",
  "environment": "production",
  "database_url": "postgresql://****:****@db.example.com:5432/judge_db"
}
```

</CodeGroup>

<CodeGroup>

```bash Connection Failed
curl -X GET https://api.getpique.ai/health/database
```

```json Response
{
  "status": "unhealthy",
  "environment": "production",
  "database_url": "postgresql://****:****@db.example.com:5432/judge_db",
  "error": "Connection timeout: unable to reach database server"
}
```

</CodeGroup>
