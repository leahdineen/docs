## Overview

The `video_matching_description` detector analyzes video content against a provided text description to determine if the video matches the expected content. It uses AI vision models to understand video content and compare it against descriptive text.

## Configuration

<ParamField body="config.content_description" type="string" required>
  Text description of expected video content to match against
</ParamField>

## Example Configuration

```json
{
  "detector_name": "video_matching_description",
  "config": {
    "content_description": "Person demonstrating makeup application with close-up shots of face and hands applying cosmetic products"
  }
}
```

## Description Guidelines

For best results, provide descriptions that include:

- **Primary activity or action** - What the person is doing
- **Setting or environment** - Where the content takes place  
- **Visual elements** - Objects, products, or props that should be visible
- **Camera work** - Shot types, angles, or framing details
- **Key moments** - Specific actions or demonstrations to look for

## Interpreting Results

The video_matching_description detector compares video content against text descriptions and returns:

- **Overall Similarity** - Matching score from 0.0 to 1.0 indicating how well the video matches the description
- **Key Elements Detected** - Array of description elements found in the video content
- **Missing Elements** - Description components not detected in the video
- **Confidence Score** - Detection confidence for the overall content match
- **Content Analysis** - Breakdown of detected activities, settings, camera angles, and objects
