## Video Quality Detectors

### video_specs

Validates video technical specifications.

<ParamField body="config.min_width" type="integer">
  Minimum video width in pixels
</ParamField>

<ParamField body="config.max_width" type="integer">
  Maximum video width in pixels
</ParamField>

<ParamField body="config.min_height" type="integer">
  Minimum video height in pixels
</ParamField>

<ParamField body="config.max_height" type="integer">
  Maximum video height in pixels
</ParamField>

<ParamField body="config.allowed_aspect_ratios" type="array">
  List of allowed aspect ratios (e.g., ["16:9", "9:16", "1:1"])
</ParamField>

<ParamField body="config.min_duration_sec" type="number">
  Minimum video duration in seconds
</ParamField>

<ParamField body="config.max_duration_sec" type="number">
  Maximum video duration in seconds
</ParamField>

<ParamField body="config.min_fps" type="number">
  Minimum frames per second
</ParamField>

<ParamField body="config.max_fps" type="number">
  Maximum frames per second
</ParamField>

**Example:**
```json
{
  "detector_name": "video_specs",
  "config": {
    "min_width": 1920,
    "min_height": 1080,
    "min_duration_sec": 15,
    "max_duration_sec": 60,
    "min_fps": 24
  }
}
```

## Content Matching Detectors

### video_matching

Matches video against reference content. **Requires preprocessing.**

<ParamField body="config.reference_videos" type="array" required>
  Array of GCS URIs of reference videos (gs://bucket/path/to/video.mp4)
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

**Example:**
```json
{
  "detector_name": "video_matching",
  "config": {
    "reference_videos": [
      "gs://judge-artifacts/org_123/reference_uploads/abc/video1.mp4",
      "gs://judge-artifacts/org_123/reference_uploads/def/video2.mp4"
    ]
  }
}
```

### audio_matching

Matches audio against reference content. **Requires preprocessing.**

<ParamField body="config.reference_audio" type="string" required>
  GCS URI of reference audio (gs://bucket/path/to/audio.mp3)
  
  **Allowed extensions:** mp3, wav, aac, flac
</ParamField>

### splitscreen_video_matching

Matches split-screen video content. **Requires preprocessing.**

<ParamField body="config.source_video_references" type="array">
  Array of GCS URIs of source video references
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

<ParamField body="config.source_video_description" type="string">
  Text description of source video
</ParamField>

<ParamField body="config.bottom_content_references" type="array">
  Array of GCS URIs of bottom content references
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

<ParamField body="config.top_content_description" type="string">
  Text description of top content
</ParamField>

**Note:** At least one of `source_video_references` or `bottom_content_references` must be provided.

### video_matching_description

Matches video against text description.

<ParamField body="config.content_description" type="string" required>
  Text description of expected content
</ParamField>

## Detectors Without Configuration

The following detectors work with default settings and don't require configuration:

- **audio_noise** - Detects background noise and audio quality issues
- **background** - Analyzes background elements
- **content_quality** - Evaluates overall content quality
- **framing** - Evaluates shot composition and framing
- **image_sharpness** - Detects blurry or out-of-focus images
- **lighting** - Analyzes lighting quality

These can be included with an empty config object:

```json
{
  "detector_name": "audio_noise",
  "config": {}
}
```
