---
title: 'Get Judge Version'
api: 'GET /judges/{judge_id}/versions/{version_number}'
description: 'Retrieve specific version details for a judge'
---

## Overview

Retrieves detailed information about a specific version of a judge, including complete performance metrics, criteria configurations, and deployment history.

## Path Parameters

<ParamField path="judge_id" type="string" required>
  Unique identifier of the judge
</ParamField>

<ParamField path="version_number" type="integer" required>
  Version number to retrieve details for
</ParamField>

## Query Parameters

<ParamField query="include_criteria_details" type="boolean" default="true">
  Include detailed criteria configurations
</ParamField>

<ParamField query="include_performance_breakdown" type="boolean" default="true">
  Include detailed performance breakdown by evaluation type
</ParamField>

## Response

<ResponseField name="judge_id" type="string" required>
  ID of the judge
</ResponseField>

<ResponseField name="version_number" type="integer" required>
  Version number
</ResponseField>

<ResponseField name="version_id" type="string" required>
  Unique version identifier
</ResponseField>

<ResponseField name="status" type="string" required>
  Version status (active, deployed, draft, archived)
</ResponseField>

<ResponseField name="criteria_configuration" type="array">
  Criteria settings for this version (if include_criteria_details=true)
</ResponseField>

<ResponseField name="performance_metrics" type="object" required>
  Comprehensive performance metrics
</ResponseField>

<ResponseField name="deployment_history" type="array" required>
  History of deployments for this version
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Version creation timestamp
</ResponseField>
