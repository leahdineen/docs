## Overview

The `sentiment` detector analyzes transcript content to identify emotional tone and sentiment. It uses OpenAI to identify specific portions that match desired or undesired sentiment criteria and detects prohibited content types.

**Requires at least one configuration parameter** - At least one of `desired_sentiment`, `undesired_sentiment`, or `prohibited_content_types` must be provided.

## Configuration

<ParamField body="config.desired_sentiment" type="array">
  List of desired sentiments to detect (e.g., ["positive", "enthusiastic", "excited"])
</ParamField>

<ParamField body="config.undesired_sentiment" type="array">
  List of undesired sentiments to flag (e.g., ["negative", "angry", "frustrated"])
</ParamField>

<ParamField body="config.prohibited_content_types" type="array">
  List of prohibited content types to detect (e.g., ["hate_speech", "inappropriate_language"])
</ParamField>

## Example Configuration

```json
{
  "detector_name": "sentiment",
  "config": {
    "desired_sentiment": ["positive", "enthusiastic"],
    "undesired_sentiment": ["negative", "angry"],
    "prohibited_content_types": ["hate_speech", "profanity"]
  }
}
```

## Interpreting Results

The sentiment detector analyzes transcript content for emotional tone and returns:

- **Desired Matches** - Array of text segments that match the desired sentiment criteria with explanations
- **Undesired Matches** - Array of text segments that match undesired sentiment patterns
- **Prohibited Content Matches** - Array of text segments containing prohibited content types
- **Sentiment Analysis** - Overall emotional tone assessment of the content
- **Match Score** - Ratio of desired sentiment matches to total analyzed content
