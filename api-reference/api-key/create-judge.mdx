title: 'Create Judge (API Key)'
api: 'POST /api-key/judges'
description: 'Create a judge using API key authentication'
---

## Overview

Creates a new judge using API key authentication for programmatic access. This endpoint provides the same functionality as the user-authenticated judge creation but uses organization API keys.

## Headers

<ParamField header="X-API-Key" type="string" required>
  Organization API key for authentication
</ParamField>

## Request Body

<ParamField body="name" type="string" required maxLength="200">
  Name of the judge
</ParamField>

<ParamField body="description" type="string" required maxLength="2000">
  Detailed description of what this judge evaluates
</ParamField>

<ParamField body="content_type" type="string" required>
  Type of content this judge evaluates (text, video, audio, image)
</ParamField>

<ParamField body="criteria" type="array" required minLength="1" maxLength="50">
  List of evaluation criteria
</ParamField>

## Response

<ResponseField name="judge_id" type="string" required>
  Unique identifier for the created judge
</ResponseField>

<ResponseField name="name" type="string" required>
  Judge name
</ResponseField>

<ResponseField name="status" type="string" required>
  Judge status (ready, learning, training)
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Creation timestamp
</ResponseField>

## Examples

<CodeGroup>

```bash Request
curl -X POST https://api.judge.example.com/api-key/judges \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "API Content Judge",
    "description": "Evaluates content quality via API",
    "content_type": "text",
    "criteria": [
      {
        "name": "Quality",
        "description": "Overall content quality"
      }
    ]
  }'
```

```json Success Response (201)
{
  "judge_id": "jdg_api_123",
  "name": "API Content Judge",
  "status": "ready",
  "created_at": "2024-01-15T15:00:00Z"
}
```

</CodeGroup>
