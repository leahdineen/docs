Each detector analyzes specific aspects of content and can be configured with custom parameters. Click on any detector below for detailed configuration options and result schemas.

## Content Matching Detectors

### video_matching

Matches video against reference content. **Requires preprocessing.**

<ParamField body="config.reference_videos" type="array" required>
  Array of GCS URIs of reference videos (gs://bucket/path/to/video.mp4)
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

**Example:**
```json
{
  "detector_name": "video_matching",
  "config": {
    "reference_videos": [
      "gs://judge-artifacts/org_123/reference_uploads/abc/video1.mp4",
      "gs://judge-artifacts/org_123/reference_uploads/def/video2.mp4"
    ]
  }
}
```

### audio_matching

Matches audio against reference content. **Requires preprocessing.**

<ParamField body="config.reference_audio" type="string" required>
  GCS URI of reference audio (gs://bucket/path/to/audio.mp3)
  
  **Allowed extensions:** mp3, wav, aac, flac
</ParamField>

### splitscreen_video_matching

Matches split-screen video content. **Requires preprocessing.**

<ParamField body="config.source_video_references" type="array">
  Array of GCS URIs of source video references
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

<ParamField body="config.source_video_description" type="string">
  Text description of source video
</ParamField>

<ParamField body="config.bottom_content_references" type="array">
  Array of GCS URIs of bottom content references
  
  **Allowed extensions:** mp4, mov, avi, mkv, webm
</ParamField>

<ParamField body="config.top_content_description" type="string">
  Text description of top content
</ParamField>

**Note:** At least one of `source_video_references` or `bottom_content_references` must be provided.

### video_matching_description

Matches video against text description.

<ParamField body="config.content_description" type="string" required>
  Text description of expected content
</ParamField>

## Text Analysis Detectors

### language

Validates content language.

<ParamField body="config.accepted_languages" type="array" required>
  List of accepted language codes (e.g., ["en", "es", "fr"])
</ParamField>

### transcript_policy

Checks transcript against policy guidelines.

<ParamField body="config.talking_points" type="array">
  List of required talking points
</ParamField>

<ParamField body="config.language_to_avoid" type="array">
  List of prohibited words or phrases
</ParamField>

<ParamField body="config.min_talking_points" type="integer">
  Minimum number of talking points required
</ParamField>

<ParamField body="config.max_language_to_avoid" type="integer">
  Maximum allowed violations
</ParamField>

### transcript_product

Detects product mentions.

<ParamField body="config.brand_name" type="string">
  Brand name to detect
</ParamField>

<ParamField body="config.product_name" type="string">
  Product name to detect
</ParamField>

### sentiment

Analyzes emotional tone.

<ParamField body="config.desired_sentiment" type="array">
  List of desired sentiments (e.g., ["positive", "enthusiastic"])
</ParamField>

<ParamField body="config.undesired_sentiment" type="array">
  List of undesired sentiments (e.g., ["negative", "angry"])
</ParamField>

<ParamField body="config.prohibited_content_types" type="array">
  List of prohibited content types
</ParamField>

## Visual Analysis Detectors

### overlays

Detects text overlays and graphics.

<ParamField body="config.disallowed_overlays" type="array">
  List of overlay categories that should NOT appear (e.g., ["text_overlay", "sticker", "watermark", "graphic"])
</ParamField>

<ParamField body="config.required_overlays" type="array">
  List of specific overlay descriptions that MUST appear (e.g., ["text showing #ad", "heart emoji"])
</ParamField>

### wardrobe

Analyzes clothing and appearance.

<ParamField body="config.wardrobe_rules" type="array">
  List of wardrobe requirements
</ParamField>

<ParamField body="config.beauty_rules" type="array">
  List of beauty/appearance requirements
</ParamField>

<ParamField body="config.max_violations" type="integer">
  Maximum allowed violations
</ParamField>

### product_application

Detects product usage demonstrations.

<ParamField body="config.require_application" type="boolean">
  Whether product application demonstration is required
</ParamField>

### subtitles

Detects subtitle presence.

<ParamField body="config.require_subtitles" type="boolean">
  Whether subtitles are required
</ParamField>

### image_ip_infringement

Checks for potential IP violations.

<ParamField body="config.allowed_terms" type="array">
  List of allowed trademarked terms
</ParamField>

## Detectors Without Configuration

The following detectors work with default settings and don't require configuration:

- **audio_noise** - Detects background noise and audio quality issues
- **background** - Analyzes background elements
- **content_quality** - Evaluates overall content quality
- **framing** - Evaluates shot composition and framing
- **image_sharpness** - Detects blurry or out-of-focus images
- **lighting** - Analyzes lighting quality

These can be included with an empty config object:

```json
{
  "detector_name": "audio_noise",
  "config": {}
}
```
