## Overview

Checks the health of all critical services including database connectivity, environment configuration, and service availability. Useful for monitoring and deployment health checks.

## Response

<ResponseField name="status" type="string" required>
  Overall system health status
  - `healthy` - All services operational
  - `unhealthy` - One or more services experiencing issues
</ResponseField>

<ResponseField name="timestamp" type="datetime" required>
  Current timestamp in ISO 8601 format
</ResponseField>

<ResponseField name="version" type="string" required>
  API version number
</ResponseField>

<ResponseField name="services" type="object" required>
  Individual service health status
  
  <Expandable title="properties">
    <ResponseField name="database" type="string" required>
      Database connectivity status (`healthy` or `unhealthy`)
    </ResponseField>
    
    <ResponseField name="openai" type="string" required>
      OpenAI service status (`healthy` or `unhealthy`)
    </ResponseField>
    
    <ResponseField name="file_storage" type="string" required>
      File storage service status (`healthy` or `unhealthy`)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="stats" type="object">
  Additional statistics (empty in basic health check)
</ResponseField>

## Examples

<CodeGroup>

```bash Success Response
curl -X GET https://api.judge.example.com/health
```

```json Response

  "timestamp": "2024-01-15T10:30:00Z",
```
