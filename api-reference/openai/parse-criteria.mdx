title: 'Parse Criteria'
api: 'POST /openai/parse-criteria'
description: 'Parse natural language into structured evaluation criteria'
---

## Overview

Uses OpenAI to parse natural language descriptions into structured evaluation criteria. This endpoint helps convert human-readable requirements into the formal criteria structure needed for judge creation.

## Request Body

<ParamField body="description" type="string" required maxLength="5000">
  Natural language description of evaluation criteria
</ParamField>

<ParamField body="content_type" type="string" required>
  Type of content these criteria will evaluate
  - `text` - Text content
  - `video` - Video content
  - `audio` - Audio content
  - `image` - Image content
</ParamField>

<ParamField body="parsing_options" type="object">
  Options for criteria parsing
  
  <Expandable title="parsing options">
    <ParamField body="max_criteria_count" type="integer" default="10" min="1" max="20">
      Maximum number of criteria to extract
    </ParamField>
    
    <ParamField body="include_weights" type="boolean" default="true">
      Whether to suggest relative importance weights
    </ParamField>
    
    <ParamField body="include_thresholds" type="boolean" default="true">
      Whether to suggest scoring thresholds
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="parsed_criteria" type="array" required>
  Extracted criteria from the description
  
  <Expandable title="criterion properties">
    <ResponseField name="name" type="string" required>
      Suggested criterion name
    </ResponseField>
    
    <ResponseField name="description" type="string" required>
      Detailed description of what this criterion evaluates
    </ResponseField>
    
    <ResponseField name="suggested_weight" type="number">
      Suggested relative importance (0.0 to 1.0)
    </ResponseField>
    
    <ResponseField name="suggested_thresholds" type="object">
      Suggested scoring thresholds
      
      <Expandable title="threshold properties">
        <ResponseField name="excellent" type="integer">
          Threshold for excellent rating (typically 90-100)
        </ResponseField>
        
        <ResponseField name="good" type="integer">
          Threshold for good rating (typically 70-89)
        </ResponseField>
        
        <ResponseField name="fair" type="integer">
          Threshold for fair rating (typically 50-69)
        </ResponseField>
        
        <ResponseField name="poor" type="integer">
          Threshold for poor rating (typically 0-49)
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="examples" type="array">
      Example scenarios for this criterion
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="parsing_confidence" type="number" required>
  Confidence in the parsing accuracy (0.0 to 1.0)
</ResponseField>

<ResponseField name="suggestions" type="array">
  Suggestions for improving the criteria
</ResponseField>

<ResponseField name="tokens_used" type="integer" required>
  Number of OpenAI tokens consumed
</ResponseField>

## Examples

```bash Request
curl -X POST https://api.getpique.ai/openai/parse-criteria \
  -H "X-API-Key: sk_org_xxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "I need to evaluate blog posts for quality. They should be well-written with clear structure, factually accurate, engaging for readers, and include proper grammar and spelling. The content should be original and provide value to the target audience.",
    "content_type": "text",
    "parsing_options": {
      "max_criteria_count": 6,
      "include_weights": true,
      "include_thresholds": true
    }
  }'
```
